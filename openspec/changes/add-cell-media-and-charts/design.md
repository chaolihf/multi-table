## Context

当前应用已支持基础文本、数字、公式和单元格样式，但缺少：
1. 富媒体内容（链接、图片）支持
2. 数据可视化能力
3. 数据管理功能（筛选、排序）

需要在现有数据结构基础上扩展单元格类型，添加图表渲染和数据操作功能。

## Goals / Non-Goals

**Goals:**
- 单元格支持链接和图片存储与渲染
- 根据选中区域生成柱状图、折线图、饼图
- 支持按列值筛选和排序
- 保持与现有功能的兼容性

**Non-Goals:**
- 不支持在线协作编辑图表
- 不支持复杂图表类型（如散点图、雷达图）
- 不支持多条件组合筛选

## Decisions

### 1. 单元格数据类型扩展
**决策**: 在 Cell 接口中添加 `link?: string` 和 `image?: string` 字段
**理由**: 最小化数据结构变更，link 存储 URL，image 存储 Base64 或 URL

### 2. 图表库选择
**决策**: 使用 Recharts 库
**理由**: React 原生支持，与现有技术栈一致，API 简洁

### 3. 图表显示方式
**决策**: 图表在独立面板/对话框中显示，不嵌入单元格
**理由**: 图表通常较大，独立显示便于交互和查看

### 4. 筛选排序实现
**决策**: 在 Canvas 渲染层实现视觉筛选/排序，不修改底层数据
**理由**: 便于撤销操作，保持数据完整性

### 5. 图片存储
**决策**: 小图片使用 Base64 存储，大图片使用 URL 引用
**理由**: 平衡性能和存储需求

## Risks / Trade-offs

**风险**: Base64 图片会显著增加 localStorage 大小
**缓解**: 限制 Base64 图片大小（如 100KB），超过则提示使用 URL

**风险**: 大量数据排序/筛选可能影响性能
**缓解**: 使用虚拟滚动，限制可视区域渲染

**风险**: 图表数据与源数据同步复杂
**缓解**: 源数据变化时提示用户刷新图表

## Migration Plan

1. 扩展 Cell 数据类型
2. 实现链接、图片渲染
3. 添加图表组件
4. 实现筛选排序逻辑
5. 更新工具栏和菜单
