## Context

本项目旨在构建一个基于 React 的多维表格应用，采用 Canvas 渲染实现高性能表格展示。需要支持公式计算、实时协作编辑和权限控制。

**技术栈约束：**
- 前端：React 18+
- 渲染：HTML5 Canvas
- 实时通信：WebSocket
- 公式解析：需要类 Excel 函数支持

**关键挑战：**
- Canvas 渲染需要处理大量单元格的绘制性能
- 实时协作需要解决操作冲突和状态同步
- 公式引擎需要支持依赖追踪和增量计算

## Goals / Non-Goals

**Goals:**
- 实现流畅的 Canvas 表格渲染（支持 1000+ 行 × 50+ 列）
- 支持常用 Excel 公式（SUM、AVERAGE、IF、VLOOKUP 等）
- 实现多人实时协同编辑（操作延迟 < 100ms）
- 提供细粒度权限控制（单元格级别的可见/编辑权限）
- 完整的单元格操作（选择、复制、粘贴、删除、清除）

**Non-Goals:**
- 不支持移动端触摸操作（首期仅支持桌面端）
- 不支持导入/导出 Excel 文件（后续迭代）
- 不支持图表、数据透视表等高级功能
- 不实现宏和脚本功能

## Decisions

### 1. Canvas 渲染架构

**决策：** 采用分层 Canvas 渲染策略

- **网格层**：绘制表格线、行列标题
- **内容层**：绘制单元格内容（文本、数字）
- **交互层**：处理选中状态、边框高亮

**理由：**
- 分层渲染可以局部重绘，提升性能
- 避免使用 DOM 元素，减少内存占用
- 可以精确控制每个像素的渲染

**替代方案考虑：**
- 使用 DOM + CSS Grid：实现简单但大数据量性能差
- 使用 WebGL：性能更好但开发复杂度高

### 2. 公式引擎实现

**决策：** 自研轻量级公式解析器

- 基于 AST 的公式解析
- 支持单元格引用（A1、B2:C10）
- 实现常用函数：数学、统计、逻辑、文本、查找
- 依赖图追踪，增量重算

**理由：**
- 现有库（如 hot-formula-parser）功能有限
- 自研可以完全控制性能和功能
- 便于与协作编辑深度集成

**替代方案考虑：**
- 使用 Formula.js + 自定义解析器：依赖外部库
- 使用 Excel 公式引擎：过于重量级

### 3. 实时协作算法

**决策：** 采用 OT (Operational Transformation) 算法

- 每个操作转换为标准操作类型（INSERT、DELETE、UPDATE）
- 服务端作为权威源进行冲突解决
- 客户端维护操作历史缓冲区

**理由：**
- OT 算法成熟，Google Docs 验证
- 适合表格这种结构化数据
- 服务端集中解决冲突，逻辑清晰

**替代方案考虑：**
- CRDT：去中心化但内存占用高
- 锁机制：实现简单但用户体验差

### 4. 权限控制模型

**决策：** 基于 RBAC + 单元格范围授权

- 角色：管理员、编辑者、查看者
- 权限粒度：工作表级别 + 单元格区域级别
- 权限检查在服务端和客户端同时进行

**理由：**
- RBAC 模型成熟易懂
- 区域授权满足大部分协作场景
- 双重检查保证安全性

### 5. 数据结构设计

**决策：** 稀疏矩阵存储

```typescript
interface Sheet {
  rows: number;
  cols: number;
  cells: Map<string, Cell>;  // key: "A1", "B2"
  formulas: Map<string, string>;  // key: "A1", value: "=SUM(B1:B10)"
}
```

**理由：**
- 表格通常是稀疏的，节省存储空间
- Map 结构便于快速查找
- 公式和内容分离，便于计算

## Risks / Trade-offs

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Canvas 渲染性能不足 | 大数据量卡顿 | 虚拟滚动、分块渲染、Web Worker |
| 公式循环引用 | 计算死循环 | 依赖图检测、最大迭代次数限制 |
| OT 算法复杂度高 | 协作冲突 bug | 充分测试、操作日志记录 |
| WebSocket 断线 | 数据丢失 | 本地缓存、自动重连、操作队列 |
| 权限检查性能 | 大量单元格时变慢 | 权限缓存、批量检查 |

## Migration Plan

本项目为全新开发，无需迁移。

**部署策略：**
1. 第一阶段：核心渲染引擎 + 基础编辑
2. 第二阶段：公式引擎
3. 第三阶段：实时协作
4. 第四阶段：权限控制

**回滚策略：** 每个阶段独立发布，问题时可回退到上一版本
