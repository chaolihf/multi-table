## Context

当前应用已有基础的 Canvas 渲染和单元格操作功能，但缺少：
1. 多 sheet 页管理界面 - 用户无法切换或管理多个工作表
2. 完整的工具栏功能 - 现有 Toolbar 组件只有 UI，没有实际功能

需要在底部添加 sheet 页签，在顶部完善工具栏功能。

## Goals / Non-Goals

**Goals:**
- 实现底部 sheet 页标签栏，支持切换、新增、右键菜单操作
- 实现工具栏功能，可设置选中单元格的背景色、字体颜色、加粗、斜体、对齐方式
- 默认创建 3 个 sheet 页（Sheet1, Sheet2, Sheet3）

**Non-Goals:**
- 不实现 sheet 页的导入导出
- 不实现跨 sheet 的公式引用
- 不实现条件格式

## Decisions

### 1. Sheet 页签位置与结构
**决策**: 在 Canvas 容器底部固定位置添加 sheet 页签栏
**理由**: 符合 Excel 等主流表格软件的用户习惯

### 2. Sheet 右键菜单
**决策**: 使用独立的 ContextMenu 组件，支持：新增、移动、复制、隐藏、显示、重命名、删除
**理由**: 复用现有右键菜单模式，保持一致性

### 3. 工具栏与 Store 集成
**决策**: 工具栏通过 `setCellStyle` 方法修改选中单元格的样式
**理由**: 与现有 `setCell` 方法分离，便于撤销重做管理

### 4. Sheet 数据结构扩展
**决策**: 在 Sheet 接口中添加 `hidden?: boolean` 字段表示隐藏状态
**理由**: 最小化数据结构变更

## Risks / Trade-offs

**风险**: 工具栏频繁调用 setCell 可能影响性能
**缓解**: 使用批量更新或防抖处理

**风险**: 隐藏 sheet 后数据持久化需特殊处理
**缓解**: 持久化时保留 hidden 状态
