## Context

当前 CanvasRenderer 组件存在 4 个交互 bug：
1. 双击单元格不会进入编辑状态
2. 单元格选中状态切换后键盘输入不会进入编辑状态
3. 复制、粘贴、插入等操作以鼠标位置下的单元格为对象，而非选中的单元格
4. 选中行、列头不会选中整行和整列

代码分析显示：
- `handleMouseDown` 中双击检测 (`e.detail === 2`) 存在但未正确触发
- `handleKeyDown` 中键盘输入逻辑与 `editingCell` 状态检查冲突
- 右键菜单操作函数使用 `selection` 但部分操作实际使用了错误的基准点
- 行/列标题点击逻辑存在但条件判断有误

## Goals / Non-Goals

**Goals:**
- 修复双击进入编辑模式
- 修复键盘输入触发编辑模式
- 修复右键菜单操作以选中区域为基准
- 修复行/列标题选择逻辑

**Non-Goals:**
- 不修改现有数据结构
- 不添加新功能
- 不修改 Canvas 渲染逻辑

## Decisions

### 1. 双击编辑修复
**决策**: 在 `handleMouseDown` 中检测双击后直接调用 `setEditingCell` store 方法，而非仅设置本地状态。
**理由**: `setEditingCellLocal` 只设置组件内状态，但编辑框渲染需要 store 中的 `editingCell` 状态同步。

### 2. 键盘输入修复
**决策**: 修改 `handleKeyDown` 逻辑，当检测到可打印字符时，先设置 `editingCell` 再让输入框捕获后续输入。
**理由**: 当前逻辑在设置编辑状态前就阻止了默认行为，导致输入丢失。

### 3. 右键菜单操作修复
**决策**: 所有右键菜单操作函数统一使用 `selection.anchor` 和 `selection.focus` 计算操作范围。
**理由**: 确保操作始终基于选中区域，而非鼠标点击位置。

### 4. 行/列选择修复
**决策**: 修正 `handleMouseDown` 中行列标题检测的坐标计算逻辑。
**理由**: 当前逻辑在计算点击位置时未正确考虑滚动偏移。

## Risks / Trade-offs

**风险**: 修改事件处理逻辑可能影响其他交互（如拖动选择）
**缓解**: 保持现有拖动逻辑不变，仅修复双击和键盘输入路径

**风险**: 键盘输入修复可能与浏览器默认行为冲突
**缓解**: 仅在选中单元格且非特殊键时拦截输入
